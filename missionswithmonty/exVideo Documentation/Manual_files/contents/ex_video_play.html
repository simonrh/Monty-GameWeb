ex_video_play

Loads and plays a video

Syntax

ex_video_play(filename, loop)
Argument Description
filename The video filename, Included File or YouTube URL (HTML5-only)
loop Whether the video will loop. true = loop, false = play once
overrideMutePolicy Whether the video overrides the browser's autoplay mute policy
Returns true if the video can play or false otherwise

Description

Loads and plays a video. The loop argument determines whether the video will loop. true = loop, false = play once. The video must have been imported as an Included File in the project, otherwise you may supply a YouTube URL (HTML5-only).

NOTE: As of 2018, many browsers have started changing their policies on media autoplay in order to combat autoplaying ad videos and other annoying media. This causes an unwanted side effect for HTML5 apps and games that want to autoplay videos with sound. The policy dictates that videos without sound can still autoplay fine but videos that contain sound must be user-initiated (e.g. user interacts with the webpage first, clicks on video, presses a keyboard key to start the video).
With the latest exVideo update, all videos will play muted by default, however if you're initiating the video via a mouse click or keyboard key you can set the parameter overrideMutePolicy to true and the video will play with sound but if you try to use it in e.g. Create event then the video will fail to play. For more info see: https://developers.google.com/web/updates/2017/09/autoplay-policy-changes

Example

//Load and play the Included File video "test.mp4" once
ex_video_play(working_directory+"test.mp4", false, false);

//Load and play a YouTube video once (HTML5-only)
ex_video_play("https://www.youtube.com/watch?v=nxOE21BtpBI", false, false);

Minimum Supported Version

exVideo 1.0.0
